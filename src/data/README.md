# src/data

This directory holds the committed JSON fixtures consumed at module-import time by `src/app/page.tsx`. Files here are static, version-controlled, and loaded synchronously — no runtime fetch, no environment dependencies. Treat them as the source-of-truth dataset for the scheduler.

## `activities.json` — COMPLETE FIXTURE

The current `activities.json` is the complete action-plan fixture: 116 records total,
with 102 primary activities and 14 backup-only fallback templates. Primary activity
priorities occupy 1..102; fallback templates occupy 103..116. The prompt/run record
for the generation pass lives at `docs/prompts/003-activities.md`.

When editing this file, keep the fixture tests in `src/lib/scheduler.test.ts` green:
schema validity, primary distribution, unique priorities, backup integrity, role
vocabulary, and canonical scheduling outcomes are all enforced there.

## `availability.json` — COMPLETE FIXTURE

Unlike `activities.json`, `availability.json` is a **complete fixture from day one**.
